# Setup & First Steps

> A utility-first CSS framework: instead of writing custom CSS classes, you compose a design directly in your HTML using small, single-purpose utility classes (`bg-red-500`, `w-52`, `rounded-full`...). The tradeoff: your HTML gets more "cluttered" with classes, but you stop context-switching between HTML and CSS files, and you never have to invent class names again.

---

## 📋 Prerequisites

Before starting, you should already be comfortable with:

- Basic **HTML** structure (tags, attributes, nesting)
- Basic **CSS** concepts (box model, flexbox/grid basics, selectors) — Tailwind doesn't replace this knowledge, it just gives you a faster way to _apply_ it.

---

## 🧰 Step 1 — Environment Setup

Make sure you have the following tools installed and configured. Each one saves you real time later, so don't skip the extensions.

### Chrome Browser

Used for live preview + DevTools inspection of your utility classes (very useful for seeing which class produced which computed style).

### VS Code + Extensions

|Extension|Why you need it|
|---|---|
|**Live Server**|Auto-refreshes the browser every time you save a file. No manual reloading.|
|**Tailwind CSS IntelliSense**|Autocomplete for class names, shows the actual CSS value on hover, and flags invalid classes.|
|**Inline Fold**|Collapses long class strings visually so your HTML stays readable while you work.|

### Node.js + npm

Tailwind is a Node-based build tool, so you need Node.js installed. Then, inside your project folder, install Tailwind as a **dev dependency**:

```bash
npm install -D tailwindcss@3.4.17 postcss autoprefixer
```

Then generate the default config file:

```bash
npx tailwindcss init
```

This creates a `tailwind.config.js` file in your project root — this is where you'll later customize colors, fonts, breakpoints, and add plugins.

> 💡 **Note:** `-D` (or `--save-dev`) means these packages are only needed during development/build, not in production — Tailwind compiles everything into a plain CSS file at the end, so the final output has zero dependency on Node.

## 🗂️ Step 2 — Project Structure & Configuration

A minimal Tailwind project looks like this:

```text
project/
├── build/
│   └── index.html          ← your actual HTML page
├── src/
│   └── input.css           ← Tailwind's "entry point" CSS file
└── tailwind.config.js       ← Tailwind configuration
```

### `tailwind.config.js`

The `content` array tells Tailwind **where to look** for class names, so it knows what to include in the final compiled CSS (Tailwind only ships the utilities you actually use — this is called "tree-shaking" / purging).

```js
/** @type {import('tailwindcss').Config} */
module.exports = {
  content: ['./build/*.html'],
  theme: {
    extend: {},
  },
  plugins: [],
}
```

> ⚠️ If you add new HTML files or move them outside `build/`, update this `content` path — otherwise Tailwind won't detect the classes you're using in them, and they'll be missing from your compiled CSS.

### `src/input.css`

This is the raw file **you** write in. It contains three special directives that Tailwind replaces with real CSS during the build step:

```css
@tailwind base;       /* resets + sensible defaults (like a modern normalize.css) */
@tailwind components; /* reusable component classes (mostly used by plugins) */
@tailwind utilities;   /* all the utility classes: bg-*, w-*, flex, etc. */
```

### Fixing the "Unknown at rule @tailwind" warning

VS Code's built-in CSS linter doesn't recognize `@tailwind` by default and will underline it as an error. To silence this:

`File → Preferences → Settings → search "unknown at rule" → under CSS: Lint → Unknown At Rules → set to "ignore"`

## 🏗️ Step 3 — Building (Compiling) Your CSS

Tailwind doesn't work directly off `input.css` in the browser — you need to **compile** it into a real, plain CSS file that your HTML links to.

### One-time build

```bash
npx tailwindcss -i ./src/input.css -o ./build/css/style.css
```

- `-i` → input file (your source)
- `-o` → output file (the compiled CSS, this is what you link in `<link>`)

### Watch mode (recommended while developing)

```bash
npx tailwindcss -i ./src/input.css -o ./build/css/style.css --watch
```

This keeps running in the background and **recompiles automatically** every time you save a change — combine this with Live Server for instant feedback.

### First example: a styled circle

```html
<div class="bg-red-500 w-52 h-52 rounded-full shadow-2xl"></div>
```

|Class|Effect|
|---|---|
|`bg-red-500`|background-color (red, shade 500 out of 50–950)|
|`w-52` / `h-52`|width & height (52 = 13rem, based on Tailwind's spacing scale)|
|`rounded-full`|border-radius: 9999px → turns a square into a circle|
|`shadow-2xl`|large drop shadow|

### Centering it on the page

Add a couple of utilities to the `<body>` itself to center any content on the page using CSS Grid:

```html
<body class="min-h-screen grid place-content-center">
    <div class="bg-red-500 w-52 h-52 rounded-full shadow-2xl"></div>
</body>
```

|`min-h-screen`|body takes at least the full viewport height|
|`grid`|turns body into a grid container|
|`place-content-center`|centers the grid's content both horizontally and vertically|

## 🎨 Step 4 — Custom Classes

Sometimes Tailwind's built-in utilities can't produce the exact effect you need (e.g. a radial gradient with specific colors). In that case, write a **plain custom CSS class** directly inside `input.css`, below the three `@tailwind` directives:

```css
@tailwind base;
@tailwind components;
@tailwind utilities;

.radial-red {
    background: radial-gradient(lightyellow, red);
}
```

Then use it like any other class in your HTML:

```html
<div class="radial-red w-52 h-52 rounded-full shadow-2xl"></div>
```

> 💡 Custom classes still get processed by the build command (`npx tailwindcss -i ... -o ...`) — they just live in your source file instead of being generated by Tailwind itself. This is the escape hatch for anything utilities can't express.

## 🧠 Quick Recap / Cheatsheet

| Task                    | Command / Class                                               |
| ----------------------- | ------------------------------------------------------------- |
| Install Tailwind        | `npm install -D tailwindcss @tailwindcss/postcss postcss`     |
| Generate config         | `npx tailwindcss init`                                        |
| One-time compile        | `npx tailwindcss -i ./src/input.css -o ./build/css/style.css` |
| Watch & auto-compile    | `... --watch` (add flag to command above)                     |
| Center anything on page | `min-h-screen grid place-content-center` on `<body>`          |
| Make a circle           | `rounded-full` + equal `w-*` / `h-*`                          |
| Add custom CSS          | Write plain CSS below the 3 `@tailwind` lines in `input.css`  |

## ❓ Common Gotchas

- **Classes not applying?** Double check the `content` path in `tailwind.config.js` actually matches where your HTML files live.
- **Nothing updates after saving?** Make sure the `--watch` process is still running in your terminal — it stops if you close the tab.
- **Forgot to link the compiled file?** Your HTML must `<link>` to the **output** file (`build/css/style.css`), never directly to `input.css`.
