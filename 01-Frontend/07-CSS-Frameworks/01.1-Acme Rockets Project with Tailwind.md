## 📦 Setting Up the Project (`package.json`)

```json
"devDependencies": {
  "autoprefixer": "^10.5.2",
  "postcss": "^8.5.19",
  "prettier-plugin-tailwindcss": "^0.8.0",
  "tailwindcss": "^3.4.17"
},
"scripts": {
  "tailwind": "npx tailwindcss -i ./src/input.css -o ./build/css/style.css --watch",
  "prettier": "npx prettier --write '**/*.html'"
}
```

- **`tailwindcss`** — the framework itself.
- **`postcss`** — the tool Tailwind runs on top of to process CSS.
- **`autoprefixer`** — automatically adds browser-specific prefixes (like `-webkit-`) so your CSS works across older browsers, without you writing them by hand.
- **`prettier-plugin-tailwindcss`** — automatically sorts your utility classes into a consistent order every time you save/format a file. You never have to think about "what order should I write these classes in?" again.
- **`scripts.tailwind`** — lets you type `npm run tailwind` instead of the full build command, and watches for changes live.
- **`scripts.prettier`** — lets you type `npm run prettier` to auto-format and reorder classes across all your HTML files.

## ⚙️ Customizing Tailwind (`tailwind.config.js`)

```js
theme: {
  extend: {
    screens: {
      widescreen: { raw: "(min-aspect-ratio: 3/2)" },
      tallscreen: { raw: "(max-aspect-ratio: 13/20)" },
    },
    keyframes: {
      "open-menu": {
        "0%": { transform: "scaleY(0)" },
        "80%": { transform: "scaleY(1.2)" },
        "100%": { transform: "scaleY(1)" },
      },
    },
    animation: {
      "open-menu": "open-menu 0.5s ease-in-out forwards",
    },
  },
},
```

### Custom breakpoints based on screen shape

Normally, Tailwind's responsive prefixes (`sm:`, `md:`, `lg:`) only care about **width**. Here, two custom breakpoints are added that care about **aspect ratio** instead:

- `widescreen` → true on very wide screens (like an ultrawide monitor).
- `tallscreen` → true on very tall, narrow screens (like a phone held upright).

Once added to the config, you use them exactly like any built-in breakpoint:

```html
<section class="widescreen:section-min-height tallscreen:section-min-height">
```

### Custom animations

Tailwind lets you define your own reusable CSS animations in two steps:

- **`keyframes`** → describes the motion itself. Here, `open-menu` grows an element from invisible (`scaleY(0)`) to slightly too big (`scaleY(1.2)`) and settles at normal size (`scaleY(1)`) — a small springy "pop open" effect.
- **`animation`** → turns those keyframes into a usable class (`animate-open-menu`) with a duration (`0.5s`), a timing curve (`ease-in-out`), and `forwards` (keep the last frame's look instead of resetting).

Now, anywhere in your HTML, one class plays the whole animation:

```html
class="animate-open-menu"
```

## 🎨 Custom Classes (`input.css`)

```css
@layer utilities {
    .section-min-height {
        min-height: calc(100vh - 68px);
    }

    .toggle-btn {
        @apply [&>div::before]:translate-y-0 [&>div::before]:rotate-45
               [&>div::after]:translate-y-0 [&>div::after]:-rotate-45
               [&>div]:bg-transparent [&>div]:rotate-[720deg];
    }
}
```

### `@layer utilities`

Wrapping custom classes in this block tells Tailwind to treat them **exactly like its own built-in utilities** — they get purged correctly if unused, and they can still be combined with responsive prefixes like `sm:` or `dark:`.

### `.section-min-height`

A simple custom class for a value Tailwind doesn't provide by default: full viewport height, minus 68px for a fixed header, so sections don't cause extra unwanted scrolling.

### `@apply`

Lets you bundle several utility classes into one reusable class name, instead of repeating a long list of classes directly in your HTML every time. `.toggle-btn` is defined once here, then simply added/removed as a single class in the markup.

### Arbitrary variants — `[&>div::before]:...`

Tailwind normally only has shorthand prefixes for common cases (`hover:`, `sm:`, `dark:`...). When you need something more specific, square brackets let you write **any CSS selector** directly:

- `&` = the element this class is applied to
- `>div` = a direct child `<div>` of that element
- `::before` = that child's `::before` pseudo-element
- everything after the last `:` is the utility applied to that exact target

So `.toggle-btn` reaches three levels deep — button → child div → that div's pseudo-elements — to turn a 3-line hamburger icon into an "X", all in one class, with no JavaScript needed for the visual change itself.

### Arbitrary values — `rotate-[720deg]`

Square brackets also let you use an exact value that isn't part of Tailwind's default scale — here, spinning a full 720° for a playful flourish, since Tailwind's built-in rotate options don't go that high.

## 🖥️ Building the Page (`index.html`)

### The Body & Dark Mode

```html
<body class="min-h-screen bg-slate-50 dark:bg-black dark:text-white">
```

- `min-h-screen` → the page is always at least full viewport height.
- `dark:bg-black` / `dark:text-white` → these classes only apply when the user's device is set to dark mode. Tailwind checks this automatically — no toggle switch needed in your code.

### Sticky Header

```html
<header class="sticky top-0 z-10 text-white bg-teal-700">
```

- `sticky top-0` → the header scrolls normally until it hits the top of the screen, then stays there.
- `z-10` → keeps it layered above other content instead of being covered by it.

### Centered Content Container

```html
<section class="flex items-center justify-between max-w-4xl p-4 mx-auto">
```

- `max-w-4xl` + `mx-auto` → the standard Tailwind way to center a block of content and cap its width, so it doesn't stretch edge-to-edge on large screens.
- `flex items-center justify-between` → puts children in a row, vertically centered, with space pushed between them (logo on one side, nav on the other).

### The CSS-Only Hamburger Icon

```html
<button id="hamburger-button" class="relative w-8 h-8 text-3xl cursor-pointer md:hidden">
  <div class="absolute top-4 -mt-0.5 h-1 w-8 rounded bg-white transition-all duration-500
    before:absolute before:h-1 before:w-8 before:-translate-x-4 before:-translate-y-3 before:rounded before:bg-white before:content-['']
    after:absolute after:h-1 after:w-8 after:-translate-x-4 after:translate-y-3 after:rounded after:bg-white after:content-['']">
  </div>
</button>
```

This builds a classic 3-bar hamburger icon with just one `<div>` and no image:

- The `div` itself is the **middle** bar.
- `before:` creates the **top** bar (pulled up with `-translate-y-3`).
- `after:` creates the **bottom** bar (pushed down with `translate-y-3`).
- `content-['']` is required — a pseudo-element won't render at all without a `content` value, even an empty one.
- `transition-all duration-500` makes any future change to these bars (like a rotation) animate smoothly instead of snapping instantly.
- `md:hidden` → this button is only shown on small screens; a full nav menu replaces it from `md:` upward.

### Mobile Menu Panel

```html
<section id="mobile-menu" class="absolute flex-col justify-center hidden w-full text-5xl origin-top bg-black top-68 animate-open-menu">
```

- `hidden` → starts closed.
- `origin-top` → makes the panel grow downward from the top edge (paired with the `scaleY` animation), like a drawer unrolling.
- `animate-open-menu` → plays the custom animation from the config the moment it becomes visible.

### Hero Section

```html
<section id="hero" class="flex flex-col-reverse items-center justify-center gap-8 p-6 mb-12 sm:flex-row">
```

- `flex-col-reverse` → on mobile, stacks children in a column but shows them in reverse order — so an image can appear above text visually, even though the text comes first in the actual HTML (good for accessibility/SEO while still controlling the visual layout).
- `sm:flex-row` → switches to a side-by-side layout once there's more screen width.
- `gap-8` → adds even spacing between items, cleaner than manually adding margin to each one.

```html
<h2 class="max-w-md text-4xl font-bold text-center sm:text-left sm:text-5xl">
```

Responsive typography: smaller and centered on mobile, bigger and left-aligned once there's room.

```html
<img class="w-1/2" src="./img/rocketdab.png" alt="Rocket Dab" />
```

`w-1/2` sizes the image to exactly half its container's width using Tailwind's fraction-based sizing.

### Product Cards

```html
<ul class="flex flex-col items-center gap-8 mx-auto my-12 list-none sm:flex-row">
```

`list-none` removes default bullet styling, since this list is being used for layout, not an actual bulleted list.

```html
<li class="flex flex-col items-center w-2/3 px-2 py-6 bg-white border border-solid shadow-xl rounded-3xl sm:w-5/6">
```

- `border border-solid` → combine to form a full border (width + style); pair with a color class to complete it.
- `rounded-3xl` → large rounded corners, giving the "card" appearance.
- `shadow-xl` → adds a drop shadow — Tailwind offers a full scale from subtle to dramatic (`shadow-sm` through `shadow-2xl`).

```html
<p class="hidden mt-2 text-3xl text-center sm:block">$</p>
<p class="mt-2 text-2xl text-center sm:hidden">Affordable Exploration</p>
```

A responsive swap: a short symbol shows on larger screens, a fuller description shows on mobile instead — not just resizing the same text, but showing genuinely different content per screen size.

### Testimonials — Decorative Quote Marks with Pseudo-Elements

```html
<blockquote class="relative py-12 pr-8 bg-teal-600 rounded-3xl pl-14">
```

`relative` sets this element as the anchor point for the decorative pseudo-elements positioned inside it.

```html
<p class="before:absolute before:top-0 before:left-0 before:text-9xl before:opacity-25 before:content-['\201C']
          after:absolute after:-bottom-20 after:right-0 after:text-9xl after:opacity-25 after:content-['\201D']">
```

- `before:content-['\201C']` generates a giant opening quotation mark using CSS, positioned in the top-left corner.
- `after:content-['\201D']` does the same for a closing quotation mark, bottom-right.
- `opacity-25` fades them to 25% so they read as a background decoration rather than competing with the actual text.
- No extra HTML elements were needed — the decoration lives entirely in CSS.

### Contact Form

```html
<button class="w-48 p-3 text-white bg-teal-700 rounded-xl hover:bg-teal-600 active:bg-teal-500">Submit</button>
```

A three-step color ramp purely through state variants:

- Normal color: `bg-teal-700`
- `hover:bg-teal-600` → slightly lighter when the mouse hovers over it
- `active:bg-teal-500` → lighter still at the exact moment it's being clicked

No JavaScript required for this kind of interactive feedback.

### Footer

```html
<nav class="flex-col hidden gap-2 md:flex" aria-label="footer">
```

The same show/hide pattern used for the hamburger menu earlier, just applied to a different piece of content and a different breakpoint (`md:` instead of `sm:`) — a reminder that you choose the breakpoint based on what actually needs the room, not just picking one size for the whole page.

## ⚙️ Making It Interactive (`main.js`)

```js
const initApp = () => {
    const hamburgerBtn = document.getElementById('hamburger-button')
    const mobileMenu = document.getElementById('mobile-menu')

    const toggleMenu = () => {
        mobileMenu.classList.toggle('hidden')
        mobileMenu.classList.toggle('flex')
        hamburgerBtn.classList.toggle('toggle-btn')
    }

    hamburgerBtn.addEventListener('click', toggleMenu)
    mobileMenu.addEventListener('click', toggleMenu)
}

document.addEventListener('DOMContentLoaded', initApp)
```

This script does no styling itself — it only turns classes on and off, and lets the Tailwind classes already in the HTML/CSS handle every visual effect. This is a simple, powerful pattern to remember: **JavaScript decides _when_ something changes, CSS decides _what it looks like_.**

- `classList.toggle('hidden')` / `'flex'` → switches the mobile menu between invisible and shown, re-triggering its open animation each time.
- `classList.toggle('toggle-btn')` → adds or removes the custom class that morphs the hamburger icon into an "X" (from the `@apply` section above).
- Clicking the menu itself (not just the button) also closes it — handy so tapping a link automatically closes the menu, without needing a separate close button.
- `DOMContentLoaded` → makes sure the script waits until the page has fully loaded before trying to find these elements.

## 🧠 Quick Reference — Concepts Covered Here

| Concept                         | Class / Syntax                                | What it does                                               |
| ------------------------------- | --------------------------------------------- | ---------------------------------------------------------- |
| Centered container              | `max-w-4xl mx-auto`                           | Caps width and centers a block horizontally                |
| Sticky element                  | `sticky top-0 z-10`                           | Sticks to the top of the viewport while scrolling          |
| Responsive breakpoints          | `sm:`, `md:`                                  | Applies a class only from a certain screen width upward    |
| Custom aspect-ratio breakpoints | `screens: { name: { raw: '...' } }`           | Breakpoints based on screen shape, not just width          |
| Dark mode                       | `dark:*`                                      | Applies only when the device is set to dark mode           |
| Show/hide toggle                | `hidden md:block` + `md:hidden`               | Classic responsive swap between two elements, no JS needed |
| Reversed layout order           | `flex-col-reverse`                            | Visually reorders elements without changing the HTML       |
| Fractional sizing               | `w-1/2`, `w-2/3`, `w-5/6`                     | Percentage-based widths using simple fractions             |
| Custom animation                | `keyframes` + `animation` in config           | Reusable named CSS animation, applied with one class       |
| Custom class                    | `@layer utilities`                            | Adds your own class that behaves like a built-in utility   |
| Bundling classes                | `@apply`                                      | Combines multiple utilities into one reusable class name   |
| Arbitrary variants              | `[&>div::before]:...`                         | Targets a specific nested element/pseudo-element directly  |
| Arbitrary values                | `rotate-[720deg]`                             | Uses an exact value outside the default scale              |
| Decorative pseudo-elements      | `before:content-[...]`, `after:content-[...]` | Adds decorative content with no extra HTML                 |
| Interaction states              | `hover:*`, `active:*`                         | Visual feedback for mouse interaction, no JS needed        |
| Class sorting                   | `prettier-plugin-tailwindcss`                 | Automatically keeps class order consistent on save         |
