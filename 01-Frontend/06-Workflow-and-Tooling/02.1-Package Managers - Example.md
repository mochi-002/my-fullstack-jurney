# A Simple To-Do List with `moment` and `uniqid`

This example shows npm managing two small, focused front-end libraries:

- **moment** — formats and displays dates/times nicely (a `dependency`)
- **uniqid** — generates a unique ID for each item we create (a `dependency`)

We'll build a tiny **To-Do List** page: every time you add a task, it gets a unique ID and a nicely formatted timestamp showing when it was created.

---

## Step 1: Create the Project Folder

```bash
mkdir todo-app
cd todo-app
```

## Step 2: Initialize `package.json`

```bash
npm init --yes
```

## Step 3: Install `moment`

```bash
npm install moment --save
```

`moment` helps us turn a raw JavaScript `Date` into something readable, like `"July 5, 2026 3:45 PM"` instead of `Sat Jul 05 2026 15:45:02 GMT+0300`.

## Step 4: Install `uniqid`

```bash
npm install uniqid --save
```

`uniqid` generates a short, unique string ID every time it's called — handy for giving each to-do item its own identifier, without writing that logic ourselves.

`package.json` now looks like this:

```json
{
  "name": "todo-app",
  "version": "1.0.0",
  "description": "A simple to-do list using moment and uniqid",
  "main": "index.html",
  "dependencies": {
    "moment": "^2.30.1",
    "uniqid": "^5.4.0"
  }
}
```

## Step 5: Create the HTML Page

Create `index.html`:

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>To-Do List</title>
</head>
<body>
  <h1>My To-Do List</h1>

  <input type="text" id="task-input" placeholder="Enter a task...">
  <button id="add-btn">Add Task</button>

  <ul id="task-list"></ul>

  <!-- Pulling both libraries straight from node_modules -->
  <script src="node_modules/moment/min/moment.min.js"></script>
  <script src="node_modules/uniqid/index.js"></script>
  <script src="script.js"></script>
</body>
</html>
```

> Note: `uniqid` is written for Node.js, so for a real production app you'd normally bundle it (e.g. with Webpack/Vite) rather than link it directly in the browser. For this simple learning example, though, linking it straight from `node_modules` is enough to see it work.

## Step 6: Create the JavaScript

Create `script.js`:

```js
const input = document.getElementById('task-input');
const addBtn = document.getElementById('add-btn');
const list = document.getElementById('task-list');

addBtn.addEventListener('click', () => {
  const taskText = input.value.trim();
  if (!taskText) return;

  // Generate a unique ID for this task
  const id = uniqid();

  // Format the current time nicely
  const timestamp = moment().format('MMMM Do YYYY, h:mm:ss a');

  // Build and display the task
  const li = document.createElement('li');
  li.id = id;
  li.textContent = `${taskText} (added: ${timestamp}) [id: ${id}]`;
  list.appendChild(li);

  input.value = '';
});
```

Every time you click **Add Task**:

- `uniqid()` generates something like `lz3k9f2h0000a1b2c3d4`
- `moment().format(...)` turns the current time into a readable string like `July 5th 2026, 3:45:02 pm`

## Step 7: Run the Project

Since this is just static HTML/JS with no dev server installed, you can open `index.html` directly in your browser, or use any simple local server you have (like VS Code's Live Server extension) to view it at `http://localhost`.

---

## Recap: Commands Used in This Example

|Command|Purpose|
|---|---|
|`npm init --yes`|Scaffold `package.json` with defaults|
|`npm install moment --save`|Install a library for formatting dates/times|
|`npm install uniqid --save`|Install a library for generating unique IDs|

**The key takeaway:** npm isn't just for big frameworks. Sometimes it's just about pulling in small, well-tested utility libraries — like `moment` for dates or `uniqid` for IDs — so you don't have to write and maintain that logic yourself.
